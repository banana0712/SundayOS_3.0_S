# SundayOS 架构设计文档

## LLM 多供应商架构

SundayOS 支持四大 LLM 供应商，通过统一的 OpenAI 兼容接口无缝切换：

```
                    ┌──────────────────────┐
                    │   SundayOS 后端       │
                    │   LLMService         │
                    │   (OpenAI SDK)       │
                    └──────────┬───────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
    ┌─────────▼────┐  ┌───────▼──────┐  ┌──────▼──────┐
    │ Ling Studio  │  │  通义千问    │  │   OpenAI    │
    │ (蚂蚁,免费)  │  │ (阿里,免费)  │  │  (付费)     │
    │ 50万token/天 │  │ 2000次/天    │  │ 按量付费    │
    └──────────────┘  └──────────────┘  └─────────────┘
```

切换方式：修改 `.env` 中 `LLM_PROVIDER` 一行即可，无需改任何代码。

## 记忆系统详解

### 四层记忆架构

| 层级 | 名称 | 存储 | 内容 | 生命周期 |
|------|------|------|------|---------|
| L1 | 工作记忆 | Redis/内存 | 当前对话上下文 | 会话级 |
| L2 | 情景记忆 | PostgreSQL | 结构化事件、对话摘要 | 永久（可衰减） |
| L3 | 语义记忆 | ChromaDB | 偏好、知识、关系 | 永久 |
| L4 | 程序性记忆 | PostgreSQL | 工作流、自动化规则 | 永久（后期） |

### 记忆衰减算法

```
effective_importance = base_importance × decay_factor^(days/30) × (1 + 0.1 × access_count)
```

- 30天半衰期
- 频繁访问的记忆衰减更慢
- CRITICAL 级别的记忆不会被自动归档

### 记忆巩固流程

1. 每晚定时触发（或达到阈值时）
2. 计算记忆间相似度
3. 合并高度相似的记忆
4. 归档低重要度的过期记忆
5. 提取新的语义知识

## 数据流

### 对话流程

```
用户输入
  │
  ├── 提取用户画像上下文
  ├── 搜索相关记忆
  ├── 获取对话历史（工作记忆）
  │
  ▼
构建系统提示词
  │
  ▼
OpenAI GPT-4o-mini
  │
  ├── 生成回复
  ├── 检测情绪
  │
  ▼
返回回复
  │
  ├── 保存对话到工作记忆
  ├── 异步提取记忆
  ├── 异步更新用户画像
  │
  ▼
用户收到回复
```

### 记忆存储流程

```
对话结束
  │
  ▼
LLM 提取记忆片段
  │
  ├── 判断记忆类型
  ├── 评估重要度
  ├── 生成标签
  │
  ▼
生成向量嵌入
  │
  ▼
按类型存入对应存储层
  │
  ▼
定期触发记忆巩固
```

```python
    "description": "技能描述",
    "parameters": { ... }
}

# 实现处理函数
async def _handle_new_skill(self, args: dict) -> str:
    # 实现逻辑
    return result
```

### 切换 LLM

修改 `.env` 中的配置即可：
```env
OPENAI_API_KEY=your-key
OPENAI_MODEL=gpt-4o  # 或 gpt-4o-mini, gpt-4-turbo
```

也可以修改 `llm_service.py` 中的 `base_url` 切换到兼容 API（如 DeepSeek、通义千问等）。
