# 11 · 评估体系

> 不依赖单一成功率，采用多维评估向量（AI Agent Systems, 2026 的 6 维 ~19 指标）+ 参与度指标（XiaoIce CPS）。

## 11.1 19 指标评估向量

设任务集 D={1..N}，轨迹 τᵢ 有 kᵢ 步，验证器返回 sᵢ∈{0,1}，标量分 rᵢ，墙钟 Tᵢ，token cᵢⁱⁿ/cᵢᵒᵘᵗ。

### 维度 1 · 端到端（主）
| 指标 | 公式 |
|------|------|
| SuccessRate | (1/N)·Σsᵢ |
| 平均分 r̄ | (1/N)·Σrᵢ |
| 平均耗时 T̄ | (1/N)·ΣTᵢ |
| 平均步数 k̄ | (1/N)·Σkᵢ |

### 维度 2 · 效率与成本
| 指标 | 公式 |
|------|------|
| 平均 token | mean(cⁱⁿ+cᵒᵘᵗ) |
| 平均成本 | mean(α·cⁱⁿ + β·cᵒᵘᵗ) |
| 平均工具调用 | mean(tool_calls) |
| p95 延迟 | Quantile₀.₉₅(T) |

### 维度 3 · 工具使用正确性
| 指标 | 公式 |
|------|------|
| 工具选择准确率 | Σ1{âₜ=aₜ*} / Σkᵢ |
| 参数/执行成功率 | 正确调用 / 总调用 |
| 失败恢复率 | ≥1 次工具失败后仍成功的任务 / 有失败的任务 |

### 维度 4 · 轨迹/规划质量
| 指标 | 公式 |
|------|------|
| 有效动作率 ValidActRate | 有效动作 / 总动作 |
| 循环率 LoopRate | mean(1 − uniq(τ)/k) |
| 计划一致性 PlanAdh | 与参考计划的逐步匹配 |

### 维度 5 · 鲁棒性
| 指标 | 公式 |
|------|------|
| 扰动下成功率 | mean over perturbations |
| 最差成功率 | min over perturbations |
| 跨种子方差 | Variance across seeds |

### 维度 6 · 安全/合规
| 指标 | 公式 |
|------|------|
| 违规率 | 策略违规 / 总动作 |
| 干预率 | 需人工干预的任务 / 总任务 |
| 每步干预数 | 干预次数 / 总步数 |

**目标值**：成功率 >85%、p95 <3s、成本 <$0.05/task、工具准确率 >90%、恢复率 >70%、有效动作率 >95%、循环率 <3%、扰动成功率 >75%、违规率 <0.1%、干预率 <5%。

## 11.2 参与度指标（个人助手核心）

| 指标 | 定义 | 目标 |
|------|------|------|
| **CPS** | 单次会话平均对话轮次 | >15（初期）→ >25（成熟） |
| DAU/MAU | 日活/月活 | >40% |
| 会话频率 | 主动发起频率 | 日均 >3 次 |
| 技能采纳率 | 使用技能数/总数 | >30% |
| 信任建立指数 | 授权高自主操作比例增长 | 持续增长 |
| 情感连接深度 | 「工具→倾诉」演进比例 | >20% 深度连接 |

> CPS 是北极星：XiaoIce 从第 1 代 5 轮增长到第 6 代 23 轮（超过人类对话的 ~9 轮）。对个人助手，参与深度比任务完成率更能反映价值。

## 11.3 评估管道

- **自动回归**：每次部署前跑标准基准（借鉴 AgentBench/WebArena/ToolBench/GAIA 的场景化改编 + 你的真实场景用例）。
- **A/B 测试**：新功能先在小流量验证 CPS 变化。
- **红队测试**：定期模拟越狱/注入，测护栏。
- **用户反馈闭环**：CPS + 满意度驱动迭代优先级。
- **引擎对比**：同一评估集跑不同引擎，验证路由决策质量（选贵的是否真的更好）。
