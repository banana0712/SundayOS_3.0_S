# 01 · 愿景与个性化

## 1.1 一句话愿景

**Sunday 不是某一个任务的 AI，而是服务于所有任务的同一个心智。**

市面上的助手是「一个任务一个 bot」：写代码用 Copilot，聊天用另一个，记事用第三个。它们互不知道你是谁。Sunday 的赌注相反：**一个连续的心智**，跨越编码、生活、学习、情感四个场景，因为共享同一份记忆、同一套人格、同一组目标，才可能真正「懂你」。

这决定了一个根本架构选择：

> **身份与模型解耦。** Sunday 的身份 = Memory + Personality + Goals + Planning + Cognitive Architecture。LLM 是可随时替换的「认知引擎」。今天用 DeepSeek，明天换 Claude，Sunday 还是 Sunday —— 因为它记得你、保持着一致的性格、追踪着你的长期目标。

## 1.2 四大角色：一个心智，服务全部

| 角色 | 场景 | 主要认知路径 | 典型引擎 |
|------|------|-------------|---------|
| 🧠 **学习/思考伙伴** | 论文理解、概念澄清、想法碰撞、第二大脑 | 系统2 深度推理 + 全量记忆检索 + Experience 抽象 | Claude / Gemini（强推理） |
| 💻 **编码/开发搭档** | 写代码、设计评审、调试、SundayOS 自身开发 | 系统2 ReAct 循环 + 工具调用（GitHub/Shell/文件） | Claude Opus / GPT（强代码） |
| 📅 **生活/日程秘书** | 日程、提醒、每日简报、主动预测需求 | 系统1 快响应 + 调度器 + 中风险预算化自主 | DeepSeek / Qwen（性价比） |
| ❤️ **情绪伴侣/对话** | 倾诉、共情、陪伴、长期关系 | 系统1 + 共情计算 + 关系记忆 | 本地模型 / DeepSeek（隐私+低延迟） |

**关键洞察**：四角色不是四个独立模块，而是同一心智在不同**认知配置**下的表现。切换角色 = 切换（记忆检索范围 × 系统1/2 权重 × 引擎选择 × 人格语气 × 自主级别），而记忆、人格、目标始终共享。

## 1.3 与你的画像适配

基于你提供的偏好，SundayOS 3.0 在通用蓝图上做了如下**个性化收敛**：

### 混合多供应商，而非绑定单一模型
你明确要求：**不把 Sunday 绑定到任何单一模型或供应商**。因此认知引擎层（[03](03-cognitive-engine-layer.md)）是本设计的头号组件：一个统一抽象层，按 **复杂度 × 延迟 × 成本 × 隐私 × 可用性** 动态路由：
- 日常对话 → 低成本模型（DeepSeek/Qwen）
- 深度推理与软件工程 → Claude / GPT
- 中文任务 → DeepSeek / Qwen
- 隐私敏感 → 未来的本地模型（Ollama）

### GitHub 作为 source of truth
你的代码、配置、甚至部分结构化记忆以 **GitHub 为权威源**。Sunday 编排各连接的 AI 服务，但状态的真相在 Git 里。这带来：版本化的人格配置、可审计的记忆变更、跨设备一致性（见 [09](09-api-and-integration.md)）。

### 多端一等公民
iPhone/iPad（Shortcuts + App Intents）、桌面、云 **都是一等公民**，采用「本地轻量推理 + 云端深度推理」混合模式。iPhone 是你的主入口（延续 1.0 的 Shortcuts 方案），但 Sunday 的心智在云端连续运行。

### 成本意识
优先国内免费/低价额度（DeepSeek、Ling、Qwen）作为默认引擎，强模型仅在系统2 深度任务时按需调用 —— 这与预算化自主（[08](08-security-and-autonomy.md)）天然契合：低风险任务用便宜引擎浅规划，高风险任务才升级强模型 + 深验证。

## 1.4 差异化定位

| 维度 | 传统 AI 助手 | SundayOS 3.0 |
|------|-------------|--------------|
| 记忆 | 单次会话，会话间遗忘 | 三层递进：存储 → 反思 → 体验 |
| 模型 | 绑定单一模型 | **可替换认知引擎 + 动态路由** |
| 推理 | 单步为主 | 双系统：快思考 + 慢思考 |
| 情感 | 基础分类 | 共情计算：意图/情绪/观点全面理解 |
| 人格 | 无或模板化 | 基于记忆流演化的一致人格 |
| 主动性 | 被动响应 | 主动引导、预测需求、提前规划 |
| 安全 | 基础过滤 | 六层护栏 + 预算化自主 + 审计 |
| 真源 | 厂商云端 | **你的 GitHub + 你的服务器** |
| 场景 | 单任务工具 | **一个心智，四大角色** |

## 1.5 成功的定义

Sunday 成功与否，不看单一任务成功率，而看：
- **CPS**（单次会话对话轮次）持续增长 —— 你愿意与它深入交流的程度。
- **信任建立指数** —— 你授权它执行更高自主级别操作的比例。
- **情感连接深度** —— 从「工具使用」到「首选倾诉对象」的演进。
- 以及 [11](11-evaluation.md) 的 19 维评估向量守住质量与安全底线。

> 用户第一次打开 Sunday 时，不应觉得「这是一个后台」，而应觉得「我正在看一个懂我的心智在运行」。
