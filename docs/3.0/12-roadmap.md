## 里程碑逻辑
- **先骨架后血肉**：Phase 1 先让「一个心智」跑起来（哪怕单系统），而非一次做全四角色。
- **引擎层最先**：因为「可替换引擎」是你的头号需求，Phase 1 就落地。
- **秘书/情感角色先行**（走便宜引擎、系统1），编码/学习角色随双系统在 Phase 2 成熟。
- **每阶段可独立交付价值**，符合渐进演进原则。
