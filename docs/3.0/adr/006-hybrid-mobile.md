# ADR-006 · 本地+云端混合（iPhone）

**状态**：采纳（继承自《3.0 架构设计计划》）

## 背景
需在 iPhone 上运行，平衡隐私、延迟与功能完整性。

## 方案
A) 纯云端；B) 纯本地；C) 本地轻量 + 云端深度混合。

## 决策
选 **C**。本地轻量模型处理敏感数据与即时响应，云端强模型处理深度推理与复杂任务。

## 理由
参考 Gemma 端侧部署与实时性约束。纯云端延迟高且隐私风险大，纯本地能力受限。混合在隐私/延迟/能力间最优。**本版联动**：本地/云端选择由认知引擎层（ADR-008）的 `privacy_sensitive` 硬约束与延迟预算驱动。详见 [09](../09-api-and-integration.md) §9.2。
