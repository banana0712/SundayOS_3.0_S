# ADR-004 · 渐进式架构演进

**状态**：采纳（继承自《3.0 架构设计计划》）

## 背景
Agent 架构从单 Agent 到多 Agent 有大量选择，需定初始架构与演化路径。

## 方案
A) 直接构建完整多 Agent；B) 从单 Agent 按需演化。

## 决策
选 **B**。Phase 1 单 Agent 验证核心价值，后续按需引入路由、多 Agent、评估器。

## 理由
Anthropic 基于 25,000+ 客户经验：复杂度必须与已验证价值成正比。多 Agent 耗 10-15× token、交付周期从周变月。过早引入增加成本但未证明价值。详见 [12](../12-roadmap.md)。
