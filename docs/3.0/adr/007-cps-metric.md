# ADR-007 · CPS 作为核心参与度指标

**状态**：采纳（继承自《3.0 架构设计计划》）

## 背景
需选评估 Sunday 成功的核心指标。传统 Agent 以任务成功率为主，但个人助手价值更多体现在参与深度。

## 方案
A) 仅任务成功率；B) 仅满意度评分；C) CPS + 多维评估向量。

## 决策
选 **C**。以 CPS（单次会话对话轮次）为核心参与度指标，辅以 19 维评估向量。

## 理由
XiaoIce 实践：CPS 从第 1 代 5 轮增长到第 6 代 23 轮，远超人类对话的 ~9 轮。CPS 反映用户愿意互动的深度与持久度，比任务完成率更适合个人助手。多维向量避免单一指标偏差。详见 [11](../11-evaluation.md)。
