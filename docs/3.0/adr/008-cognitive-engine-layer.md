# ADR-008 · 认知引擎抽象层（引擎可替换）★

**状态**：采纳（本个性化设计新增，头号决策）

## 背景
用户核心要求：**「Sunday should treat AI models as interchangeable cognitive engines, not as its identity.」** 不绑定任何单一模型/供应商，需按复杂度、延迟、成本、隐私、可用性动态路由，并支持未来本地模型。1.0 已有「多供应商可切换」，但那是静态配置切换，非自动选优。

## 考虑方案
- A) 静态配置单模型（1.0 现状）：改 `.env` 切换。简单但无法按任务选优、无回退。
- B) 硬编码 if-else 选模型：能按任务选，但耦合、难扩展、无成本/可用性感知。
- C) **统一 Provider 抽象 + 动态路由器**：引擎自描述能力/成本，路由器按五维打分选优，带回退链与熔断。

## 决策
选 **C**。新增独立的 L1.5 认知引擎层：`EngineProvider` 抽象 + `CognitiveRouter`（复杂度分类 + 预算化打分 + 回退链 + 记账追踪）。新增引擎 = 写子类 + 登记能力/价格，零改上层。

## 理由
- 直接兑现用户「模型即引擎、非身份」的理念——身份留在记忆/人格/目标。
- 与预算化自主（ADR 概念）天然统一：低风险便宜引擎浅思考，高风险强引擎深验证，省钱与安全同一机制。
- 与四角色天然契合：情感/秘书走便宜引擎，编码/学习走强引擎。
- 回退链保证心智「永不死机」，优雅降级而非失败。
