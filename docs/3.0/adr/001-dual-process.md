# ADR-001 · 双系统认知架构

**状态**：采纳（继承自《3.0 架构设计计划》）

## 背景
需在响应速度与推理深度间平衡。单模型难同时满足毫秒级对话与深度多步推理。

## 方案
A) 单模型全量推理（纯 ReAct）；B) 双系统（Talker+Reasoner 分离）；C) 级联模型（先小后大）。

## 决策
选 **B**。系统1(Talker) 轻量常在线处理日常；系统2(Reasoner) 强模型按需激活处理复杂任务。

## 理由
参考 DeepMind Talker-Reasoner 与 XiaoIce：分离快慢思考实现最优资源利用与体验。A 延迟过高，C 级联失败体验差。**本版增强**：系统2 的引擎经认知引擎层（ADR-008）动态选择，而非固定绑定。详见 [05](../05-dual-process-cognition.md)。
