# 08 · 安全与自主

> 安全是系统的一等属性，嵌入执行循环，而非事后附加。融合 OpenAI 多层护栏、AutoHarness 自动护栏合成、AI Agent Systems 的预算化自主。

## 8.1 六层护栏（纵深防御）

护栏与主推理**乐观并发**执行，违规抛 tripwire 异常立即中止（借鉴 OpenAI Agents SDK）。

| 层 | 护栏 | 机制 | 时机 |
|----|------|------|------|
| L1 | 代码 Harness 自动合成 | 从工具 API 规范自动生成类型/范围/权限检查代码 | 编译时 + 部署前 |
| L2 | LLM 安全分类器 | 检测越狱、注入攻击、有害内容 | 每次 LLM 输入/输出 |
| L3 | PII 过滤器 | 检测并脱敏个人身份信息 | 数据流入/流出 |
| L4 | 规则护栏 | 长度限制、黑名单、正则、速率限制 | 请求到达 |
| L5 | 工具风险评估 | 工具分低/中/高风险，高风险暂停或审批 | 工具调用前 |
| L6 | Human-in-the-Loop | 超限重试或高风险动作升级人工确认 | 异常/高风险 |

### 输入护栏流水线（执行顺序）
```
用户输入
 → 长度/字符限制 (L4)
 → 规则/正则/黑名单 (L4)
 → Moderation (L2, 仇恨/暴力等)
 → LLM 相关性分类器 + 安全分类器 (L2, 可用 L1 便宜引擎)
 → is_safe?  否 → 拒绝「无法处理该消息」
           是 → 进入认知循环
```

### 工具风险评估（L5）
每个工具/技能按三维打分定级：是否只读、可逆性、所需权限、财务影响。

| 风险 | 示例 | 处理 |
|------|------|------|
| 低 | 查天气、读记忆、搜索 | 自动执行 |
| 中 | 发邮件、改日程、提交 GitHub | 执行 + 事后通知 |
| 高 | 删文件、大额支付、权限变更 | 执行前人工确认 |

### 验证器 V（一等组件）
Agent Transformer `A=(πθ,M,T,V,E)` 中 V 是操作语义的核心，**非可选附加**。每个 Action 在执行前经 V 校验（类型/范围/权限/风险），不通过则阻断或升级 HITL。

## 8.2 预算化自主（Budgeted Autonomy）

据任务风险与不确定性，自适应调整**规划深度 + 验证强度 + 引擎档位**。把 Agent 从「尽最大努力」变为「在约束下最优」。

| 自主级别 | 风险 | 规划深度 | 验证强度 | 人工参与 | 引擎档 | 示例 |
|---------|------|---------|---------|---------|-------|------|
| 完全自主 | 低 | 浅层 ≤3 步 | 基础 | 无 | L1-L2 便宜 | 查天气、设闹钟 |
| 半自主 | 中 | 中等 3-7 步 | 完整 + 审计 | 事后通知 | L2-L3 | 发邮件、创建日程 |
| 建议模式 | 中高 | 深度 7+ 步 | 完整 + 树搜索 | 执行前确认 | L3 强 | 改重要文件、大额购买 |
| 人工主导 | 高 | 仅分析建议 | 全量 + 护栏 | 必须人工执行 | L4 最强 | 删账户、权限变更、支付 |

**与引擎路由联动**：预算化自主直接驱动 [03](03-cognitive-engine-layer.md) 的复杂度权重——低风险任务成本权重高（用便宜引擎浅思考），高风险任务能力权重高（用最强引擎 + 深验证）。这让「省钱」与「安全」在同一个机制里统一。

**执行控制**：LLM 是「预算化循环里的规划器/控制器」，受 时间/token/工具调用次数/可执行副作用 的显式上限约束，只在任务困难或高风险时才分配「深思考」。

## 8.3 隐私保护

作为个人 AI，Sunday 访问大量私密数据，隐私必须内建：
- **数据最小化**：只收集完成任务所必需。
- **本地处理优先**：情感、健康、私人对话在设备端/本地引擎处理（对接 [03](03-cognitive-engine-layer.md) 的 `privacy_sensitive` 硬约束）。
- **E2E 加密**：记忆传输与存储全程 AES-256-GCM。
- **差分隐私**：上传云端的数据加 ε=1.0 噪声。
- **隐私感知遗忘**：随时删除特定/全部记忆（含向量库同步清除）。
- **社交圈层隔离**：朋友/同事/陌生人不同圈层访问不同粒度。
- **审计透明**：可查看 Sunday 访问了哪些数据、做了哪些决策的完整日志。

## 8.4 审计

- 不可变审计日志，记录从用户输入到工具调用的全链路（含 [03](03-cognitive-engine-layer.md) 的 RouteTrace 与 ReAct 思维轨迹）。
- Thought Editing：允许人工编辑 Agent 的 Thought 轨迹来纠正行为（比改输出更有效）。
- 每个决策可追溯到具体推理步骤。

## 8.5 密钥与真源安全 ⚠️

- **绝不硬编码密钥**。一律经 `.env` + 环境变量注入。
- **当前风险**：仓库根 `readme.txt` / `其他.txt` 含明文真实 API Key（DeepSeek、Anthropic 中转）+ 带 key 的 dashboard URL。**立即轮换**并加入 `.gitignore`。
- GitHub 作为真源时，提交前经密钥扫描（如 gitleaks）防止泄露。
- `persona.yaml`、技能定义等版本化文件不得含敏感值。
