# 开发契约 · SundayOS Engineering Contract

> **每个进来改代码的 AI，动手前必读本文件。** 不管你能力强弱。
>
> ARCHITECTURE 说的是"系统**长什么样**"；本契约说的是"你**被允许怎么改它**"。
> 这是规矩，不是建议。配合 `/checkup`（裁判）形成闭环：**契约定规矩，checkup 查违规。**
>
> 存在的理由：项目不会"崩溃"，只会**慢慢烂**。烂的根源是每个 AI 各堆各的、
> 没有统一底线。这份契约就是那条底线，让不同能力的 AI 进来都不会把项目带沟里。

---

## 0. 三条铁律（绝不可破）

1. **绝不硬编码密钥。** 一律 `.env` + 环境变量。密钥不进代码、不进文档、不进 git。
2. **高风险操作先问主人。** 删文件/删库/改权限/破坏性 git（force push / reset --hard）先确认。
3. **说"完成"之前必须真正运行验证。** 见 §3。跑通测试 ≠ 验证；要真的把 app 跑起来看。

---

## 1. 硬限制（可检查，checkup 会查）

- **单文件 ≤ 600 行。** 超了就该拆。
  - 例外记录：`backend/app/main.py`（当前 ~1442 行）、`webchat.py`（~1220 行）是**已知债务**，
    只许变小、不许变大。新功能不准再往 main.py 塞路由。
- **路由按域放 `backend/app/routers/<域>.py`**（auth / chat / conversations / memory / admin / debug），
  不再往 main.py 堆。main.py 只做装配。
- **新引擎 = 写 `EngineProvider` 子类 + 在 `registry.py` 登记，零改上层。** 这是架构第一原则，别绕过。
- **一个模块一个职责。** 一个文件干两件不相关的事 = 该拆。

## 2. 单一真相源（这次腐烂的根源，重点防）

- **子系统通过 `runtime.<name>` 访问**，不要各处开模块级全局单例。一套引用，别两套。
- **禁止同一逻辑写两份。** 血的教训：`/api/chat` 和 `/api/chat/stream` 各写一遍聊天逻辑，
  结果流式那份忘了记统计，仪表盘悄悄少算、还不报错。**要复用就抽成一个函数，别复制粘贴。**

## 3. 完成的定义（Definition of Done · 说"完成"前逐条过）

- [ ] 改了核心逻辑（路由/记忆/双系统/护栏/认证）→ **补了测试**，`pytest` 全绿
- [ ] 动了前端 → `npm run build` 通过
- [ ] **真正把 app 跑起来、在实际界面/接口验证了这次改动**（不是只跑测试、不是 import 调用函数）
- [ ] 没有把假数据 / mock / 写死的状态当"真实"展示给主人
- [ ] 文档同步：VERSION / CHANGELOG / CURRENT_STATE / PROJECT_MEMORY（按 CLAUDE.md 会话结束纪律）

**任何一条没做到，就不是"完成"，是"写了一半"。如实说。**

## 4. 改动前的纪律

- **多文件 / 不明确的改动：先设计后编码**，给方案、必要时问主人，再动手。
- **一次只做一件事，做透、验证、再下一件。** 屎山大多是"顺手多做了点"堆出来的。
- **不确定要不要做、或涉及破坏性/扩范围：先问主人。** 别自作主张扩大战线。
- 复杂或跨文件研究**优先派子 Agent**，保住主上下文。

## 5. 欠债规则（时间压力下的唯一合法退路）

赶时间必须破例（比如没拆就往 main.py 加了个路由）→ **记进 `PROJECT_MEMORY.md` 技术债表**：
什么债、为什么欠、何时还。**绝不无声欠债。** 记下来的债是可管理的；藏起来的债才会烂。

## 6. 进出协议

- **进场**：`/bootstrap` —— 对齐项目全貌，不依赖聊天历史。
- **收尾 / 自检**：`/checkup` —— 体检项目健康，给主人大白话报告。

## 7. 能力分级（给不同强弱的 AI）

- **强 AI**：可以承担大重构（拆 main.py、runtime 收口），但同样受 §0-§5 约束。
- **弱 AI**：哪怕做不了大事，也**必须守住**：三条铁律（§0）+ 完成定义（§3）+ 一次一件（§4）。
  拿不准就**缩小范围**，做一件小而完整、验证过的事，好过做一堆半成品。
- **通用**：违反本契约任何一条而不记录，就是在给项目埋雷。宁可少做，不可乱做。
