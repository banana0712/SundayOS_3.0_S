## 勘误记录

### [ERRATA-001] PROJECT_MEMORY.md §2.12 构建产物提交状态错误

**发现时间**: 2026-07-17  
**发现方式**: `/verify` 技能验证 v0.10.2 文档变更时发现  
**影响范围**: PROJECT_MEMORY.md

#### 原始错误内容

```markdown
**当前状态**：
- 已提交到 Git 并部署到服务器
- 包含 node_modules 和 .next 构建输出（.gitignore 已配置但文件已提交）
```

#### 验证过程

```bash
# 检查 .gitignore 配置
$ cat frontend/.gitignore
/node_modules
/.next/
# ✓ 已正确配置

# 检查 git 跟踪状态
$ git ls-files frontend/ | grep -E "(node_modules|\.next)" | wc -l
0
# ✓ 没有任何文件被跟踪

# 检查 git 状态
$ git status frontend/node_modules frontend/.next
nothing to commit, working tree clean
# ✓ 这些目录未被跟踪
```

**结论**: node_modules 和 .next 目录仅存在于本地工作树，从未提交到 git。

#### 修正后内容

```markdown
**当前状态**：
- 已提交到 Git 并部署到服务器
- .gitignore 已正确配置排除 node_modules 和 .next 构建输出
- 构建产物仅存在于本地工作树，未提交到版本控制
```

#### 根本原因

推测原因：文档作者（可能是 AI）看到本地存在这些目录，误以为已提交到 git。

#### 预防措施

1. 涉及 git 状态的声明必须通过 `git ls-files` 或 `git status` 验证
2. 区分"存在于工作树"和"已提交到版本控制"两个概念
3. 使用 `/verify` 技能验证文档变更时应检查关键声明的真实性

## 勘误记录规范

每条勘误记录应包含：

1. **编号**: ERRATA-XXX（按时间顺序递增）
2. **发现时间**: YYYY-MM-DD
3. **发现方式**: 人工审查 / 自动验证 / 用户报告
4. **影响范围**: 受影响的文件列表
5. **原始错误内容**: 完整引用原文（代码块）
6. **验证过程**: 如何确认这是错误（命令输出、截图等）
7. **修正后内容**: 完整引用修正后的内容（代码块）
8. **根本原因**: 为什么会出现这个错误
9. **预防措施**: 如何避免类似错误再次发生
